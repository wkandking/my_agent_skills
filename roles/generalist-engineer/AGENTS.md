# generalist-engineer

默认角色。用于日常 feature、bugfix、refactor、普通 review、轻量脚本和仓库改动。

## Use This Role When

- 任务不以机制解释为核心风险
- 任务不以性能证据和容量判断为核心输出
- 你需要推进实际开发工作或知识库中的常规改动

## Do Not Use This Role For

- 深度机制归因和原理解释
- 正式 benchmark、load、stress、capacity 结论

## Shared Load Hints

- 先读 `base/skills/`
- 再读 `base/notes/`
- 然后只补 1 个本角色 `note` 或共享 `skill`

## Skills

- 暂无角色专用 skill；默认优先复用 `base/skills/eat/` 和 `base/skills/skill-creator-codex/`

## Notes

- `notes/default-engineering-boundaries.md` — 默认使用这个角色，除非任务的核心价值在机制解释或性能证据

## Questions

- `questions.md` — 2 条待验证：默认 code review 深度、是否需要单独的 release note
