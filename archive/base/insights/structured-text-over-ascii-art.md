# 知识文档避免 ASCII 字符画，用结构化文本

## 问题

ASCII 字符画（用 `│├▼──→` 等符号画的流程图/架构图）对 agent 不友好：
- LLM 靠 token 序列理解内容，不是"看"图。空格对齐和 Unicode 符号的空间关系需要额外推理，容易误读分支层级
- 字符画信息密度低，占 context 多（一个 60 行字符画可以用 10 行结构化文本表达）
- 修改困难，加一个节点可能需要重排整个图

## 替代方案

用显式的关系列表：

```
主链路（顺序执行）：
1. A → B：说明
2. B → C：说明

分支链路：
- C → D：说明
- C → E：说明
```

对于 DAG，用链式 `→` 一行表达：
```
step_a → step_b → step_c → step_d → step_e
```

## 适用范围

- 给 agent 消费的知识文档（skills/experience/insights）
- 给人看的报告可以用图片或 Mermaid，不受此限制
