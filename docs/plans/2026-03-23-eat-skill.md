# Eat Skill Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Extend the shared `eat` skill so it can ingest explicit sources or the most recent attachment group, classify reusable knowledge into a target root, and let `/eat update` plus `/eat update all` maintain `eat` itself through proposal-first, user-approved updates.

**Architecture:** The skill lives under `base/skills/eat/`. `SKILL.md` defines the shorthand invocation model, target-root resolution, source-selection behavior, normal proposal/apply behavior, and the self-maintenance workflows for `/eat update` and `/eat update all`. `references/knowledge-placement.md` provides the durable decision table for where shared and role-specific knowledge should be stored, how to initialize missing knowledge-root structure safely, what source types are supported, and how `eat` is allowed to maintain itself. `base/AGENTS.md` continues to advertise the skill in the shared inventory.

**Tech Stack:** Markdown, repository knowledge layout conventions, Codex skill frontmatter
